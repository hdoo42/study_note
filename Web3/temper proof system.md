# Tamper-proof System

## 정의
**Tamper-proof system**은 **변조 방지 시스템**이다.  
누군가가 무단으로 시스템을 수정하거나 조작(tampering)하려 해도, 이를 어렵거나 불가능하게 만든 구조를 의미한다.  
주로 **보안성**, **신뢰성**, **무결성**을 보장하기 위해 설계된다.

---

## 특징
- **외부 조작에 강함**  
  일반적인 수단으로는 변경, 수정, 삭제 등이 어렵다.
- **물리적・논리적 보호**  
  하드웨어 차원의 봉인이나 소프트웨어 검증 체계를 갖춘다.
- **위변조 시도 기록 또는 차단**  
  변조를 시도하면 기록이 남거나 작동이 중단된다.

---

## 예시
- **하드웨어**
  - 전기・가스 검침기: 봉인을 훼손하면 기록 발생
  - 보안 카메라: 케이스를 열면 작동 중지 또는 경고
- **소프트웨어**
  - 디지털 서명된 파일: 파일 변조 여부를 검증
- **네트워크・시스템**
  - 블록체인: 데이터 변조 시 전체 체인 검증 실패

---

## 관련 개념 비교

| 개념               | 설명                                  |
|--------------------|---------------------------------------|
| Tamper-proof       | 변조 자체를 거의 불가능하게 설계        |
| Tamper-resistant   | 변조를 어렵게 만들었지만 가능성 존재     |
| Tamper-evident     | 변조를 막지는 않지만 변조 사실을 쉽게 감지 가능 |

---

## 대표 적용 분야
- **핀테크**: 결제 단말기, 인증 시스템
- **의료기기**: 환자 생체 데이터 기록 장치
- **군사・보안장비**: 통신 장비, 드론
- **블록체인**: 스마트 컨트랙트, 분산 원장 시스템
- **산업용 IoT**: 원격 센서, 모니터링 시스템

---

## 요약
> **Tamper-proof system**은 무단 변경이나 조작을 원천적으로 막아, 시스템의 신뢰성과 무결성을 유지하기 위해 필수적으로 사용된다.

# 블록체인의 Tamper-proof 한계

## 1. 51% 공격 (51% Attack)

- 블록체인은 **다수 노드(>50%)** 가 정직하게 작동한다는 가정 위에서 설계되었다.
- 만약 어떤 공격자가 네트워크의 **과반수(51% 이상)** 컴퓨팅 파워나 지분을 장악한다면:
  - 원하는 대로 새로운 블록을 생성하고,
  - 기존 거래를 취소하거나,
  - 이중 지불(double-spending)을 할 수 있다.
- 이 경우 블록체인의 **변조 방지(tamper-proof) 성질이 깨질 수 있다**.

**예시**  
→ [[PoW|PoW]] 방식에서는 채굴 컴퓨팅 파워의 51%를 장악  
→ [[PoS|PoS]] 방식에서는 전체 지분의 51%를 소유

## 2. 스마트 계약(Smart Contract) 취약점

- 블록체인 위에 작성된 **스마트 계약**은 사람이 코딩하는 것이다.
- 코드 자체에 **논리적 오류**나 **보안 취약점**이 있을 경우,
  블록체인 자체는 tamper-proof라도,  
  계약 내용은 조작되거나 악용될 수 있다.

**예시**  
→ DAO 해킹 사건(2016년): 스마트 계약 취약점을 이용해 대량의 이더리움 탈취

---

## 3. 초기 조건의 신뢰 (Initial Trust Assumptions)

- 블록체인에 기록되는 "초기 데이터"가 잘못되었다면,  
  아무리 그 이후 기록이 tamper-proof하더라도  
  **잘못된 데이터 기반으로 시스템이 굴러간다.**
- 즉, 블록체인은 데이터 "기록 이후"는 안전하지만, "기록 당시"의 진실성은 따로 보장하지 않는다.

**예시**  
→ 실물 상품 공급망 추적 블록체인: 공급자가 처음부터 허위 데이터 입력 시 문제 발생

---

## 4. 경제적 유인(Economic Incentives)

- 공격자 입장에서, 블록체인을 변조할 **경제적 이득이** 충분히 크다면  
  51% 장악을 위해 거대한 자금을 투자할 가능성도 있다.
- 이 경우 tamper-proof는 이론적으로는 유지되지만, 현실에서는 깨질 수 있다.

---

## 요약
> 블록체인은 매우 높은 수준의 tamper-proof 특성을 가지고 있지만,  
> **51% 공격**, **스마트 계약 취약점**, **초기 데이터 신뢰성**, **경제적 유인 문제**로 인해 완전히 무적은 아니다.  
> 따라서 설계・운영・모니터링을 통해 추가 보완이 필요하다.
